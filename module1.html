Имя профиля: tun1

Устройство: tun1

Удаленный IP: 172.16.2.2

[Нужен скриншот: настроенного туннеля tun1 на HQ-RTR (ip addr show tun1)]

7. Настройка динамической маршрутизации (OSPF)

Задание: Обеспечьте динамическую маршрутизацию на маршрутизаторах HQ-RTR и BR-RTR: сети одного офиса должны быть доступны из другого офиса и наоборот. Для обеспечения динамической маршрутизации используйте link state протокол на усмотрение участника: разрешите выбранный протокол только на интерфейсах IP туннеля, маршрутизаторы должны делиться маршрутами только друг с другом, обеспечьте защиту выбранного протокола посредством парольной защиты. Сведения о настройке и защите протокола занесите в отчёт.

Настройка OSPF на HQ-RTR:

dnf install -y frr

nano /etc/frr/daemons # включаем ospfd

[Нужен скриншот: включенного ospfd в /etc/frr/daemons на HQ-RTR]

systemctl start frr

vtysh

configure terminal

router ospf

ospf router-id 1.1.1.1

passive-interface default

network 10.0.0.0/30 area 0

network 192.168.100.0/27 area 0

network 192.168.200.0/27 area 0

network 192.168.99.0/29 area 0

area 0 authentication

exit

interface tun1

no ip ospf network broadcast

no ip ospf passive

ip ospf authentication

ip ospf authentication-key P@ssw0rd

exit

end

write

systemctl enable --now frr

systemctl restart frr

Настройка OSPF на BR-RTR:

network 192.168.30.0/28 area 0

area 0 authentication

exit

no ip ospf network broadcast

no ip ospf passive

ip ospf authentication

ip ospf authentication-key P@ssw0rd

end

Проверка OSPF:

[Нужен скриншот: настроенного sudoers для net_admin]

5. Безопасный удаленный доступ (SSH)

Задание: Настройте безопасный удаленный доступ на серверах HQ-SRV и BR-SRV: для подключения используйте порт 2026, разрешите подключения исключительно пользователю sshuser, ограничьте количество попыток входа до двух, настройте баннер «Authorized access only».

SSH настройка на HQ-SRV и BR-SRV:

Обязательно нужно создавать конфиг файл в sshd_config.d папке а не в ssh_config.d

nano /etc/ssh/sshd_config.d/ssh.conf

Port 2026

AllowUsers sshuser

MaxAuthTries 2

Banner /etc/ssh/banner

[Нужен скриншот: содержимого /etc/ssh/sshd_config.d/ssh.conf (порт 2026, AllowUsers sshuser)]

Создание баннера:

nano /etc/ssh/banner

Authorised access only!

[Нужен скриншот: файла баннера /etc/ssh/banner с текстом "Authorised access only!"]

Открытие порта 2026 для SELinux:

dnf install -y policycoreutils-python-utils

semanage port -a -t ssh_port_t -p tcp 2026

systemctl restart sshd

ss -tlnp | grep 2026

[Нужен скриншот: прослушивания порта 2026 (ss -tlnp | grep 2026)]

Проверка SSH с другой машины:

ssh sshuser@192.168.30.2 -p 2026

Как раз увидишь Banner

[Нужен скриншот: успешного подключения по SSH с отображением баннера]

10. Инфраструктура разрешения доменных имён

Задание: Настройте инфраструктуру разрешения доменных имён для офисов HQ и BR: основной DNS-сервер реализован на HQ-SRV, сервер должен обеспечивать разрешение имён в сетевые адреса устройств и обратно в соответствии с таблицей 3, в качестве DNS сервера пересылки используйте любой общедоступный DNS сервер (77.88.8.7, 77.88.8.3 или другие).

Настройка DNS на HQ-SRV:

dnf install -y bind bind-utils

nano /etc/named.conf

Вставляем в "options":

listen-on port 53 { 127.0.0.1; 192.168.100.2; };

allow-query { any; };

forwarders {77.88.8.8; 8.8.8.8; 1.1.1.1; };

recursion yes;

Убираем копии этих параметров в файле чтобы было по одному параметру.

В конце /etc/named.conf указываем наши зоны:

zone "au-team.irpo" IN { type master; file "/var/named/au-team.irpo.zone"; }; zone "100.168.192.in-addr.arpa" IN { type master; file "/var/named/rev.100.zone"; }; zone "200.168.192.in-addr.arpa" IN { type master; file "/var/named/rev.200.zone"; };

[Нужен скриншот: файла /etc/named.conf с настроенными зонами]

Теперь сами файлы зон:

nano /var/named/au-team.irpo.zone

$TTL 86400 @ IN SOA hq-srv.au-team.irpo. admin.au-team.irpo. ( 2026061201 3600 1800 604800 86400 ) @ IN NS hq-srv.au-team.irpo. hq-rtr IN A 172.16.1.2 br-rtr IN A 172.16.2.2 hq-srv IN A 192.168.100.2 hq-cli IN A 192.168.200.2 br-srv IN A 192.168.30.2 docker IN A 172.16.1.1 web IN A 172.16.2.1 _kerberos._tcp IN SRV 0 0 88 br-srv.au-team.irpo. _ldap._tcp IN SRV 0 0 389 br-srv.au-team.irpo.

[Нужен скриншот: файла зоны /var/named/au-team.irpo.zone]

nano /var/named/rev.100.zone

$TTL 86400 @ IN SOA hq-srv.au-team.irpo. admin.au-team.irpo. ( 2026061201 3600 1800 604800 86400 ) @ IN NS hq-srv.au-team.irpo. 2 IN PTR hq-srv.au-team.irpo. 1 IN PTR hq-rtr.au-team.irpo.

nano /var/named/rev.200.zone

$TTL 86400 @ IN SOA hq-srv.au-team.irpo. admin.au-team.irpo. ( 2026061201 3600 1800 604800 86400 ) @ IN NS hq-srv.au-team.irpo. 2 IN PTR hq-cli.au-team.irpo.

Даем права сервису прочитать наши зоны:

chown named:named /var/named/*.zone

Проверки:

named-checkconf

systemctl enable --now named

[Нужен скриншот: проверки named-checkconf и named-checkzone]
